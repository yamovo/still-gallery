## 为什么提示词很重要

用 AI 工具的人越来越多，但大多数人得到的结果平平无奇。问题不在 AI，而在输入。垃圾进，垃圾出——这个道理从来没变过。

提示词工程（Prompt Engineering）不是什么高深技术，它的核心就是**把你想清楚的事情表达清楚**。

## 几个实用原则

### 1. 说清楚你要什么

模糊的提示词得到模糊的结果。

```
❌ 写一篇关于摄影的文章
✅ 写一篇 800 字的街头摄影入门指南，面向有手机但没相机的初学者，语气轻松，包含 3 个实用技巧
```

越具体，AI 越能对准你的预期。

### 2. 给上下文

AI 不知道你是谁、在做什么、为什么需要这个。把背景说清楚：

```
我是一个个人博客的作者，网站用暗色主题，读者是对设计和技术感兴趣的人。
帮我写一篇关于字体选择的文章，重点讲衬线和无衬线字体在屏幕上的表现差异。
```

### 3. 指定格式

想要列表、表格、还是段落？想要长文还是摘要？直接说。

```
用 Markdown 格式，分 3 个要点，每个要点配一个代码示例。
```

### 4. 迭代优化

第一次的结果很少是最终版。把 AI 的输出当成初稿，继续追问：

- "第二段太抽象了，加一个具体例子"
- "语气太正式，改成聊天风格"
- "帮我把这段缩短到 200 字以内"

## 常见误区

**背诵提示词模板**：网上流传很多"万能提示词"，但真正有用的提示词是针对具体任务定制的。模板可以参考，但不要照搬。

**过度复杂化**：有些人把提示词写成一篇论文。其实简洁明了就够了，关键是信息密度，不是字数。

**忽略否定指令**：告诉 AI 不要做什么，和告诉它要做什么一样重要。比如"不要用技术术语"、"不要超过 500 字"。

## 不同场景的提示词策略

### 写作类
- 明确受众、语气、长度
- 给出大纲或结构要求
- 指定引用或例子的类型

### 编程类
- 说明技术栈和版本
- 给出输入输出的示例
- 描述边界情况

### 分析类
- 提供原始数据或背景
- 说明分析的角度和深度
- 指定输出格式（表格/图表/结论）

## 最后

提示词工程的本质不是操控 AI，而是**想清楚自己要什么**。当你能把需求描述清楚的时候，不管用什么 AI 工具，结果都不会太差。

写作如此，做事亦然。
